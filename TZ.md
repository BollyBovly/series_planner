### SeriesPlanner (Планировщик сериалов)

1. Цель проекта

Создание веб-сервиса для интеллектуального планирования просмотра сериалов. Система позволяет пользователям:

    * Добавлять сериалы в личный список

    * Указывать количество часов, доступных для просмотра в день

    * Получать подборку эпизодов на конкретный день, оптимизированную под их временные возможности

    * Видеть прогноз: сколько дней потребуется для завершения сериала при текущем темпе просмотра

    * Отслеживать прогресс и историю просмотра

**Проблема**: Люди не знают, реально ли они смогут закончить сериал за разумное время, учитывая свои ежедневные возможности.
2. Роли пользователей
Гость

    * Просмотр каталога доступных сериалов

    * Просмотр основной информации о сериалах (кол-во эпизодов, длительность)

    * Расчет примерного времени на просмотр (без сохранения)

Зарегистрированный пользователь

    * Все права гостя

    * Создание личного списка сериалов ("Мой просмотр")

    * Установка параметров просмотра (часов в день, часов в неделю)

    * Добавление информации о просмотренных эпизодах

    * Получение рекомендаций по эпизодам на день

    * Просмотр прогноза завершения сериала

    * История просмотра с датами

Администратор

    * Управление каталогом сериалов

    * Управление пользователями

    * Просмотр статистики использования сервиса

3. Модели данных (сущности)
Модель 1: Series (Сериал)

Карточка сериала с основной информацией.

    * title (CharField) – название сериала

    * description (TextField) – описание сюжета

    * total_seasons (IntegerField) – кол-во сезонов

    * total_episodes (IntegerField) – общее кол-во эпизодов

    * average_episode_duration (IntegerField) – средняя длительность эпизода в минутах

    * genres (CharField) – жанры (например: "Драма, Фантастика")

    * created_at (DateTimeField) – дата добавления

    * poster_url (URLField) – ссылка на постер (необязательно)

Модель 2: Episode (Эпизод)

Информация о каждом эпизоде.

    * series (ForeignKey на Series) – св

    * season_number (IntegerField) – номер сезона

    * episode_number (IntegerField) – номер эпизода в сезоне

    * title (CharField) – название эпизода

    * duration (IntegerField) – длительность в минутах

Модель 3: WatchingHistory (История просмотров)

Логирование активности для аналитики.

Ключевые поля:

    * user (ForeignKey на User) – пользователь

    * series (ForeignKey на Series) – сериал

    * episode (ForeignKey на Episode) – эпизод (опционально)

    * watched_at (DateTimeField) – дата и время просмотра

    * duration_watched (IntegerField) – длительность просмотра в минутах

4. Технический стек

    * Backend: Django 5.1.2

    * База данных: PostgreSQL (production), SQLite (development)

    * Frontend: Bootstrap 5.3.2, Bootstrap Icons 1.11.3
